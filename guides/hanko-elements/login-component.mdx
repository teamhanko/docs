## Use Case
The `<hanko-login>` component is ideal for building secure, user-friendly login flows in modern web applications. It enables seamless integration of various authentication methodsâ€”such as passkeys, passwords, and OAuthâ€”without the need to build your own UI or handle complex backend logic.

This component is particularly useful when you want to create a dedicated login-only page. Unlike `<hanko-auth>`, it blocks registration functionality, making it perfect for apps where registration is closed or controlledâ€”such as internal tools, enterprise apps, or invite-only platforms.

If you want to Combine Login and Register on the same page: \
[try out the`<hanko-auth>` component](/guides/hanko-elements/auth-component).

If you want to create a dedicated Registration page:  \
[try out the `<hanko-register>` component](/guides/hanko-elements/register-component) .


## Features
|  |  |
| --- | --- |
|  **- Login** | Allows users to securely log in to their existing accounts.
|  **- Passkeys** | Enables passwordless login with modern device credentials.
|  **- Passcodes** | Useful for email-based login and verification workflows.
|  **- Mobile biometrics** | Supports biometric login via mobile devices.
|  **- FIDO Security Keys** | Offers the highest level of hardware-based authentication.
|  **- OAuth SSO** | 	Allows users to log in using third-party identity providers (e.g., Google, Apple).
|  **- Passwords** | Supports traditional password-based login for broader accessibility.


## Usage

### Markup

```html 
<hanko-login></hanko-login>
```

### Attributes

| Name | Description |
| --- | --- |
| `prefilled-email` | Used to prefill the email input field |
| `lang` | Used to specify the language of the content within the element |
| `experimental` | A space-separated list of experimental features to be enabled |
